# 디자인초기설정

## 언제 쓰나
디자이너가 *처음으로* 이 프로젝트(또는 처음으로 이 컴퓨터)에서 디자이너 모드를 켤 때. 시작 *의식*이라 1회성, 이후엔 자연어("새로 만들고 싶어"/"보여줘"/"저장해줘")로 흘러간다.

## 동작

### 0. 사전 도구 확인 (가장 먼저)

디자이너 모드 작동에 필요한 도구가 다 있는지 먼저 확인한다. 다 있으면 짧은 한 줄("도구 확인 완료") 후 §1로 진행. 빠진 게 있으면 *어떻게 깔지* 한국어로 친절히 안내하고 흐름을 *멈춘다* — 사용자가 설치/설정 후 `/kdesigner:디자인초기설정`을 다시 호출하면 이어서 시작한다.

#### 0.1 검사 항목 (한 번에 다 검사 후 결과 정리)

먼저 `uname -s`로 OS 판별(`Darwin` = macOS, 그 외 = Windows/Linux). 안내 문구가 OS에 따라 갈라진다.

| 항목 | 명령 | 분류 |
|---|---|---|
| 화면 엔진 (`Node.js`) | `node --version` | 필수 (없으면 작업 자체 불가) |
| 작업 시점 저장 도구 (`git`) | `git --version` | 필수 |
| 사용자 표시 정보 | `git config --global user.name` + `git config --global user.email` | 필수 |
| 도구 설치 도우미 (`Homebrew`) | `brew --version` | macOS 한정 권장 (있으면 다른 도구 설치가 한 줄로 끝) |
| 팀 공유 공간 도구 (`gh`) | `gh --version` | 선택 (인계 단계에서 필요) |
| 팀 공유 로그인 | `gh auth status` | 선택 |

검사 결과는 *한꺼번에* 모은다 — 사용자가 한 번에 무엇을 깔지 알 수 있게(깔다 다시 호출, 또 깔다 다시 호출 흐름 방지).

#### 0.2 모든 도구 OK인 경우

응답 본문 맨 위에 짧은 한 줄(진행 단계 가시화):
> 도구 확인 완료 — 디자이너 모드 켜는 중이에요.

→ §1으로 그대로 진행.

#### 0.3 필수 도구 누락 → 흐름 정지 + 친화 안내

영어 에러는 한 줄도 노출 X. *해당하는 항목만* 묶어 한 응답에 함께 노출. macOS면 brew 경로를 우선 권유, brew도 없거나 Windows/Linux면 fallback 경로 안내.

**`Homebrew` 미설치 (macOS 한정, 권장)**:
> **도구 설치 도우미**(`Homebrew`)가 아직 없어요 — 한 줄로 다른 도구들을 깔 수 있게 해주는 macOS용 도우미라, 깔아두시면 아래 도구들 설치가 훨씬 편해져요. 공식 사이트(`https://brew.sh`) 첫 화면에 *복사해서 터미널에 붙여넣을* 한 줄 명령이 있어요 — 그대로 실행하시면 돼요(약 5–10분, 중간에 비밀번호 한 번 물어요).
>
> brew 없이도 진행은 가능해요(아래 fallback 경로). 깔지 말지는 자유롭게 결정해주세요.

**`Node.js` 미설치**:
- macOS + brew 있음: > **화면 엔진**(`Node.js`)이 깔려 있지 않아요 — 디자이너가 만든 화면을 띄우는 데 꼭 필요한 도구예요. 터미널에 `brew install node`를 입력하시면 약 1분 안에 깔려요. 끝나면 *지금 열려 있는 터미널·대화창을 닫고 다시 열어주세요* — 새로 깐 도구가 그제서야 인식돼요.
- 그 외 (brew 없음 / Windows / Linux): > **화면 엔진**(`Node.js`)이 깔려 있지 않아요 — 디자이너가 만든 화면을 띄우는 데 꼭 필요한 도구예요. 공식 사이트(`https://nodejs.org`)에서 *LTS* 버전 설치 파일을 받아 실행하시면 돼요. 설치가 끝나면 *지금 열려 있는 터미널·대화창을 닫고 다시 열어주세요* — 새로 깐 도구가 그제서야 인식돼요.

**`git` 미설치**:
- macOS + brew 있음: > **작업 시점 저장 도구**(`git`)가 깔려 있지 않아요 — "방금 작업 그대로 다시 돌아가기" 같은 안전망에 꼭 필요해요. 터미널에 `brew install git`를 입력하시면 1–2분 안에 끝나요.
- macOS + brew 없음: > **작업 시점 저장 도구**(`git`)가 깔려 있지 않아요. 터미널에 `xcode-select --install`을 한 번 입력하시면 자동 설치 창이 떠요(약 5분).
- Windows: > **작업 시점 저장 도구**(`git`)가 깔려 있지 않아요. 공식 사이트(`https://git-scm.com`) 안내를 따라가시면 돼요.

**`git` 사용자 정보 미설정** (`user.name` 또는 `user.email` 비어있음):
> **누가 만든 작업인지 표시할 정보**(이름·이메일)가 아직 없어요. 저장 시점(`commit`)마다 한 줄로 들어가요 — 한 번만 알려주시면 다시 묻지 않아요.
>
> (`AskUserQuestion`로 이름·이메일 묻기. 받은 값으로 `git config --global user.name "..."` / `git config --global user.email "..."` 실행. 사용자가 거부하면 안내 한 줄 후 흐름 중단 — 강제 X.)

#### 0.4 선택 도구 누락 → 안내만, 흐름 진행

`gh` 미설치 또는 `gh auth status` 실패:
- macOS + brew 있음: > **팀 공유 공간 연결**(`GitHub` + `gh`)은 아직 준비 안 됐어요. 지금 당장은 없어도 디자인 작업은 시작할 수 있고, 나중에 *개발자한테 넘기기* 단계에서 필요해져요. 미리 깔아두시려면 `brew install gh` 한 줄이면 돼요.
- 그 외: > **팀 공유 공간 연결**(`GitHub` + `gh`)은 아직 준비 안 됐어요. 지금 당장은 없어도 디자인 작업은 시작할 수 있고, 나중에 *개발자한테 넘기기* 단계에서 필요해져요. 미리 깔아두시려면 공식 사이트(`https://cli.github.com`) 안내 따라 설치하시면 돼요.

`gh`는 깔려 있는데 로그인만 안 된 경우:
> **팀 공유 공간**(`GitHub`) 도구는 깔려 있는데 로그인은 아직이에요. 나중에 *개발자한테 넘기기* 단계에서 필요해질 때 다시 안내드릴게요.

→ 한 줄만 본문 위에 노출, *필수 도구가 다 있다는 전제로 §1로 그대로 진행*.

#### 0.5 흐름 정지 시 응답 끝 안내

필수 도구 누락으로 흐름을 멈춘 경우 응답 마지막에:

> 위 도구들이 다 준비되면 **`/kdesigner:디자인초기설정`을 한 번 더 불러주세요** — 같은 자리에서 이어서 시작해요.

다음 행동 1개 제안(글로벌 §7) 자리는 이 안내가 대체.

### 1. 사전 점검
다음을 `Read`/`Bash test`로 빠르게 확인하고, 분기 결정에만 쓴다(중간 응답에는 노출 X):

| 항목 | 확인 |
|---|---|
| 현재 디렉토리에 `CLAUDE.project.md` 존재 | 이미 켜져 있음 → §6 "이미 시작됨" 분기 |
| `~/.claude/CLAUDE.md` 존재 + 디자이너 페르소나 마커(`<!-- kd:designer-mode -->`) | 전역 이미 켜져 있음 → §3 스킵 |
| `./CLAUDE.project.md.kd-archived-*` 발견 (하나 이상) + `./CLAUDE.project.md` 부재 | 이전 세션 작업 흔적 → §1.5 복원 분기 |
| 현재 디렉토리에 `package.json` 또는 `components/` | 기존 코드 → §5에서 `import-existing` 권유 |
| 완전 빈 디렉토리(또는 `.git`만) | §5에서 `new-service` 권유 |

### 1.5 archived 발견 시 복원 분기 (선택)

`./CLAUDE.project.md`은 *없는데* `./CLAUDE.project.md.kd-archived-*`가 하나 이상 있으면 — `/kdesigner:디자인끄기`로 끄고 다시 켜는 흐름. 이전 작업의 컴포넌트 인덱스·디자인 토큰을 살릴지 묻는다.

`AskUserQuestion`:

> 이 폴더에 *이전에 디자이너 모드로 작업한 흔적*이 있어요(`CLAUDE.project.md.kd-archived-...`). 그때 정리해뒀던 디자인 약속·컴포넌트 정리를 *이어서* 쓸 수 있어요.
>
> - **이어서 쓰기** (이전 작업 내용 그대로 살리기, 다시 분석 필요 X)
> - **새로 시작** (이전 흔적은 보관해둔 채 비어있는 상태로 새로 시작)
> - **잘 모르겠어요** → 기본값: 이어서 쓰기

선택 결과대로:
- **이어서 쓰기**:
  - `ls -t ./CLAUDE.project.md.kd-archived-* | head -1` 로 가장 최근 archive 추출
  - `mv` 로 그 파일을 `./CLAUDE.project.md`로 복원 (cp 아님 — 한 archive를 두 자리에 두지 않음)
  - 그 후 §2.2 import 라인 처리는 *복원된 파일 기준으로 그대로* 진행 (마커 검증 통과)
  - §2.1 템플릿 생성은 *스킵* (이미 `CLAUDE.project.md` 존재)
- **새로 시작**:
  - archive 파일은 *손대지 않음* (보관)
  - §2.1부터 그대로 진행 (템플릿으로 새 `CLAUDE.project.md` 생성)

응답에 복원 시 한 줄 노출:
> 이전 작업 그대로 이어서 시작해요 — `CLAUDE.project.md.kd-archived-YYYYMMDD-HHmm`을 다시 살렸어요.

### 2. 프로젝트 단위 활성화 (기본, 묻지 않음)
프로젝트 단위는 위험이 거의 없음 — 사용자 디렉토리 *내부*만 건드림. **묻지 않고** 진행.

§1.5에서 *이어서 쓰기*로 복원했으면 §2.1은 스킵하고 §2.2부터.

#### 2.1 `CLAUDE.project.md` 생성

`plugin/templates/CLAUDE.project.md`(이 플러그인의 템플릿) 내용을 그대로 가져와 현재 디렉토리 루트에 `CLAUDE.project.md`로 생성:

- `Read` 템플릿 → `Write` 프로젝트 루트에 (`./CLAUDE.project.md`)
- `{{서비스명}}`은 폴더 이름(`basename "$PWD"`)으로 채움
- 나머지 `{{...}}` placeholder는 *그대로 둔다* — `new-service`/`import-existing`이 이후 채울 영역
- 이미 존재하면 덮어쓰지 *않고* §6 "이미 시작됨" 분기

#### 2.2 `./CLAUDE.md`에 import 라인 추가 (자동 로드 보장)

`CLAUDE.project.md`는 *비표준 이름*이라 새 대화 시작 시 자동 로드되지 않는다 — 자동 로드되는 파일은 `./CLAUDE.md`, `./.claude/CLAUDE.md`, `./CLAUDE.local.md`, `~/.claude/CLAUDE.md`. 따라서 자동 로드되는 `./CLAUDE.md`에 공식 `@<path>` import 문법으로 한 줄 박아 *새 대화에서도 디자인 약속(토큰·컴포넌트 인덱스·페르소나 요약)이 그대로 이어지게* 한다.

처리 절차:

1. **`./CLAUDE.md` 존재 여부 확인** — `Read` 또는 `test -f ./CLAUDE.md`.

2-a. **없으면** (새로 생성):
   - `touch ./.kd-no-prior-claude-md` — 빈 마커 파일. `/kdesigner:디자인끄기`가 *우리 플러그인이 새로 만든 파일인지* 판단해서 통째 제거할 수 있게.
   - `Write ./CLAUDE.md`로 다음 내용만 박는다:
     ```
     <!-- kd:designer-mode:start -->
     @CLAUDE.project.md
     <!-- kd:designer-mode:end -->
     ```

2-b. **있으면** (기존 파일에 마커 격리해 추가):
   - **백업** — `cp ./CLAUDE.md ./CLAUDE.md.kd-backup-<YYYYMMDD-HHmm>` (timestamp는 `date +%Y%m%d-%H%M`). `/kdesigner:디자인끄기`에서 *원본 그대로* 복원할 수 있게.
   - **append** — 기존 파일 끝에 빈 줄 하나 + 다음 마커 격리 블록 append:
     ```
     
     <!-- kd:designer-mode:start -->
     @CLAUDE.project.md
     <!-- kd:designer-mode:end -->
     ```

3. **마커 검증** — append/생성 후 `Read ./CLAUDE.md`로 다시 읽어 시작 마커(`<!-- kd:designer-mode:start -->`)와 끝 마커(`<!-- kd:designer-mode:end -->`)가 *정확히 1쌍*인지 확인. 0쌍 또는 2쌍 이상이면 사용자에게 응답 끝에 경고 1줄 노출 ("`./CLAUDE.md`에 디자이너 마커가 정상이 아닌 상태예요 — 한 번 확인해주세요").

마커 사이는 *오직 `@CLAUDE.project.md` 한 줄만*. 디자인 시스템·페르소나 본문은 박지 않는다 — 본문은 `CLAUDE.project.md`에 있고, `@import`이 자동 로드시킨다.

### 3. 전역(user-level) 활성화 — 명시 동의 1회 (위험 동작)

CLAUDE.md §5: 전역 `~/.claude/CLAUDE.md` 수정은 *다른 모든 프로젝트*에 영향. 반드시 명시 동의.

`AskUserQuestion`:

> 디자이너 모드를 **이 컴퓨터의 모든 프로젝트**에서 자동으로 켜려면 전역 설정 파일에 한 줄 추가가 필요해요.
>
> **무엇이 바뀌나요**: `~/.claude/CLAUDE.md` 끝에 한국어 디자이너 페르소나 안내가 추가돼요(원본은 그대로 백업).
> **언제 되돌리나요**: `/kdesigner:디자인끄기` 한 번이면 원본 그대로 복원돼요.
>
> - **이 컴퓨터 전체에서 켜기** (편함, 다른 프로젝트에도 디자이너 모드 자동 적용)
> - **이 프로젝트에서만 켜기** (안전, 다른 프로젝트는 영향 X)
> - **잘 모르겠어요** → 기본값: 이 프로젝트에서만

선택 결과대로:

#### 3-A. "이 컴퓨터 전체에서 켜기" 선택 시
1. **백업** — `~/.claude/CLAUDE.md`가 있으면 `~/.claude/CLAUDE.md.kd-backup-<YYYYMMDD-HHmm>`로 `cp`. 없으면 빈 백업 마커(`~/.claude/.kd-no-prior-claude-md`) 생성.
2. **추가** — `plugin/templates/CLAUDE.user.md` 내용을 `~/.claude/CLAUDE.md` 끝에 *append*. 시작·끝 마커로 격리:
   ```
   <!-- kd:designer-mode:start -->
   ... (templates/CLAUDE.user.md 내용)
   <!-- kd:designer-mode:end -->
   ```
   `/kdesigner:디자인끄기`가 이 마커 사이만 잘라 복원할 수 있게.
3. 백업 경로를 응답에 1줄로 노출 — "원본은 `~/.claude/CLAUDE.md.kd-backup-YYYYMMDD-HHmm`에 보관해뒀어요".

#### 3-B. "이 프로젝트에서만" 또는 "잘 모르겠어요" 선택 시
- 전역 파일 *건드리지 않음*. §2의 `CLAUDE.project.md` 끝 페르소나 요약 섹션이 단독으로 디자이너 모드를 지탱(`templates/CLAUDE.project.md` §페르소나 요약).
- "필요하면 나중에 `/kdesigner:디자인초기설정`을 다시 불러서 전역으로 켤 수 있어요" 안내 한 줄.

### 4. 검증 — 마커·백업 확인

### 4.1 사이클 시작 시점 기록 (export-handoff용)

`export-handoff`가 *이번 사이클에 추가/변경된 파일만* 인계 README에 정리할 때 기준점으로 쓸 SHA를 박아둔다. 박아두지 않으면 옛 mock·옛 import까지 끌려와 개발자에게 잘못된 인계가 가게 됨.

처리 절차:

1. **git 저장소가 아니면 스킵** (`git rev-parse --git-dir 2>/dev/null` 실패).
2. **commit 0개면 스킵** (`git rev-parse HEAD 2>/dev/null` 실패).
3. 그 외:
   - `mkdir -p ./.claude`
   - **이미 `.claude/.kd-session-base` 존재 시 덮어쓰지 X** (다른 흐름이 먼저 박았을 수 있음 — 처음 박는 1회만 책임).
   - 부재 시: `echo "$(git rev-parse HEAD)" > ./.claude/.kd-session-base`
4. **`.gitignore` 처리** (자동 추가):
   - `.gitignore`에 `.claude/.kd-session-base` 한 줄이 없으면 *append* (기존 줄 보존).
   - `.gitignore` 자체가 없으면 새로 만들어 그 한 줄만 박기.
   - `.claude/` 디렉토리 자체는 gitignore X.

이 SHA는 `export-handoff`가 `git diff $BASE_SHA..HEAD`로 *이번 사이클 변경분*만 추릴 때 기준점이 된다.
