# 되돌리기

## 언제 쓰나
디자이너가 "이거 마음에 안 드는데 되돌리고 싶어"·"방금 한 거 취소"·"오전에 했던 상태로"라고 할 때. 위험 동작이라 슬래시로 *명시 트리거*. 자연어로는 발동 X (오발 방지).

## 동작

### 1. 사전 점검 (읽기만)

| 점검 | 동작 |
|---|---|
| `git rev-parse --git-dir` 실패 | git 저장소 아님 → §6 "되돌릴 게 없음" 분기 |
| `git log --oneline -1` 비어있음 | commit 0개 → §6 분기 |
| `git status --porcelain` 변경 있음 | 작업 중 변경 미저장 → §2에서 먼저 안내 |

### 2. 미저장 변경 가드
저장 안 된 변경이 있으면 revert가 충돌·덮어쓰기 위험. 먼저 묶어두거나 정리해야 안전:

> 지금 **저장 안 된 변경**이 있어요. 되돌리기 전에 정리가 필요해요.
>
> - **지금까지 한 거 먼저 저장하고 되돌리기** (안전)
> - **저장 안 된 변경은 그대로 두고 되돌리기는 취소**
> - **잘 모르겠어요** → 기본값: 먼저 저장하고 되돌리기

"먼저 저장" 선택 시 자연어로 `safe-save` Skill에 위임("작업 중간 저장" 톤). 이후 §3로.

### 3. 되돌릴 시점 선택 (질문 1/1)

`git log --oneline -10`을 `Read`/`Bash`로 받아 사용자 친화 옵션으로 가공. 영어 SHA·"commit" 단어 단독 노출 X — 메시지·시각만.

`AskUserQuestion`:

> 어디로 되돌릴까요? *되돌리기는 다시 한 번 더 되돌릴 수 있어요(안전).*
>
> - **방금 한 거 1단계 되돌리기** — "<가장 최근 commit 메시지>"
> - **그 전 단계까지 되돌리기** — "<2번째 commit 메시지>"
> - **오늘 오전 상태로** (오늘 작업 시작 시점) — "<해당 commit 메시지>"
> - **잘 모르겠어요** → 기본값: 방금 한 거 1단계만

옵션 라벨은 단순화하되, 응답 본문에 *각 시점에서 무엇이 바뀔지 한 줄*씩 첨부:

> "방금 한 거 1단계 되돌리기"는 *메인 화면 카드 색 다듬기*가 취소돼요.

각 시점 요약은 `git show --stat <sha>`의 변경 파일 수·핵심 경로로 1줄 요약(메인 모델이 한국어로 가공).

### 4. 명시 동의 — "이거 되돌립니다" 1회 (위험 동작)

§3에서 시점 선택했어도 *실행 직전* 다시 한 번 확인. CLAUDE.md §5: 위험 동작은 매번 명시 동의.

`AskUserQuestion`:

> 정말 되돌릴까요?
>
> **무엇이 바뀌나요**: <선택한 시점 요약 1줄>
> **다시 되돌릴 수 있나요**: 네 — 이 되돌리기 자체도 한 단계 작업으로 남아서, 또 한 번 되돌리면 원래대로 돌아와요.
>
> - **네, 되돌릴게요**
> - **취소**

"취소" 선택 시 §6과 동일한 톤으로 "되돌리지 않았어요" 응답.

### 5. `git revert` 실행 (비파괴)

선택된 시점에 따라 명령:

| 선택 | 명령 |
|---|---|
| 방금 한 거 1단계 | `git revert --no-edit HEAD` |
| 그 전 단계까지 | `git revert --no-edit HEAD~1..HEAD` (최근 2개 한 번에) |
| 오늘 오전 상태로 | `git revert --no-edit <오늘 첫 commit의 부모>..HEAD` |

#### 5.1 절대 사용 X (CLAUDE.md §3 / 글로벌 정책)
- `git reset --hard` ❌
- `git checkout -- .` ❌
- `git push --force` ❌
- `git clean -f` ❌
- `git revert` 시 `-n`/`--no-commit` ❌ (revert 자체를 새 commit으로 *남기는* 게 핵심)

#### 5.2 충돌 발생 시
revert 도중 conflict 발생하면 자동 해결 시도하지 *않고* `error-translator`로 위임 — "되돌리는 중 부딪힘이 있어요, 같이 풀어야 해요" 톤. 사용자가 직접 결정해야 하는 영역.

자동 회복 우선 정책(CLAUDE.md §8)이 있더라도, *revert conflict는 사용자 의도 영역*이므로 자동 해결 X. 이 한 가지는 예외.

#### 5.3 push 여부
- *원격에 이미 올라간 commit*을 되돌렸으면 → "원격에도 반영해두려면 올려야 해요" 안내. `safe-save` 호출 흐름이 아니므로 push는 사용자 다음 발화("올려줘")에 맡긴다.
- 로컬에만 있던 commit이면 push 안내 X.

### 6. "되돌릴 게 없음" / "취소" 분기

git 저장소가 아니거나 commit이 없거나 §4에서 취소 시:

> 되돌리지 않았어요 — 지금 상태 그대로예요.

`safe-save`로 한 번도 저장한 적 없으면 안내 추가:
> 한 번도 저장한 적이 없어서 되돌릴 시점도 아직 없어요. **"저장해줘"** 한 번 해두면 다음부터 이 시점으로 돌아올 수 있어요.

### 7. 결과 안내

성공 응답 패턴:
> **되돌렸어요** — *<선택한 시점 요약>* 부분이 취소됐어요. 이 되돌리기 자체도 한 단계로 남아 있어서, 다시 한 번 되돌리면 원래대로 와요.

응답에 SHA·"revert"·"commit" 단어 단독 노출 X — 한국어 의미만.

## 안전 장치

- **명시 동의 2단계** — 시점 선택(§3) + 실행 직전 확인(§4). 위험 동작이라 매번 1회 명시 동의(CLAUDE.md §5, PRD §6).
- **비파괴 절대 원칙** — `git reset --hard`/`push --force`/`clean -f`/`checkout --` 모두 X. 모든 되돌림은 `git revert` 새 commit으로(CLAUDE.md §3).
- **revert 자체가 되돌릴 수 있음** — `git revert <revert-commit>`로 또 되돌릴 수 있다는 점을 사용자에게 1줄로 안내 → 안심 효과.
- **미저장 변경 가드** — 작업 중 미저장이 있으면 먼저 저장 권유, *덮어쓰기 위험* 회피.
- **conflict는 자동 해결 X** — 사용자 의도 영역이라 `error-translator`로 위임만, 자동 결정 X.
- **자연어 발동 X** — 슬래시로만 호출. "되돌려줘"는 자연어로도 들어올 수 있지만 그 경우에도 이 슬래시 흐름을 *권유만* 하고 자동 실행은 슬래시 명시 호출 시에만(오발 방지).
- **외부 노출(push) 자동 X** — revert 결과를 원격에 올리는 것은 사용자 다음 발화("올려줘")에 맡김.
