# 디자인끄기

## 언제 쓰나
디자이너 모드를 더 이상 켜두지 않고 싶을 때. (a) 이 프로젝트에서만 빼거나 (b) 다른 컴퓨터·다른 프로젝트에 영향 안 주게 *전역 설정까지 원래대로* 복원할 때.

## 동작

### 1. 현재 상태 진단 (읽기만)
어디까지 켜져 있는지부터 본다 — 끄는 범위가 달라지므로:

| 점검 | 의미 |
|---|---|
| `./CLAUDE.project.md` 존재 | 프로젝트 단위 켜져 있음 |
| `./CLAUDE.md`에 `<!-- kd:designer-mode:start -->`/`end` 마커 | 프로젝트 루트 import 라인 박혀 있음 |
| `./CLAUDE.md.kd-backup-*` 백업 파일 | 프로젝트 루트 `CLAUDE.md` 복원 가능 |
| `./.kd-no-prior-claude-md` 마커 | 원래 `./CLAUDE.md`가 *없었음* (끄면 파일 자체 제거) |
| `~/.claude/CLAUDE.md`에 `<!-- kd:designer-mode:start -->`/`end` 마커 | 전역 켜져 있음 |
| `~/.claude/CLAUDE.md.kd-backup-*` 백업 파일 | 전역 복원 가능 |
| `~/.claude/.kd-no-prior-claude-md` 마커 | 원래 `~/.claude/CLAUDE.md`가 *없었음* (전역 끄면 파일 자체 제거) |

전역도 프로젝트도 켜져 있지 않으면 §6 "이미 꺼져 있음" 분기로.

### 2. 끄는 범위 결정 (질문 1/1)

`AskUserQuestion`. CLAUDE.md §5 — "왜 묻는지" 1줄 + "잘 모르겠어요" 옵션.

질문은 §1 진단 결과에 따라 옵션이 달라진다:

#### 2-A. 프로젝트만 켜져 있는 경우
> 디자이너 모드를 끄는데, **이 프로젝트만** 영향이 있어요.
>
> - **이 프로젝트에서 끄기** (`CLAUDE.project.md` 정리)
> - **잘 모르겠어요** → 기본값: 이 프로젝트에서 끄기

#### 2-B. 전역도 켜져 있는 경우
> 디자이너 모드를 어디까지 끌까요? *왜 묻냐면, 다른 프로젝트에 영향이 갈 수 있어서요.*
>
> - **이 프로젝트만 끄기** (`CLAUDE.project.md`만 정리, 다른 프로젝트는 그대로 켜져 있음)
> - **이 컴퓨터 전체에서 끄기** (`~/.claude/CLAUDE.md` 원본 복원, 다른 프로젝트도 영향)
> - **둘 다 끄기** (이 프로젝트 + 전역 — 가장 깔끔)
> - **잘 모르겠어요** → 기본값: 이 프로젝트만 끄기 (가장 안전)

### 3. 프로젝트 단위 끄기

선택지가 *이 프로젝트만 끄기* 또는 *둘 다 끄기*면 진행.

#### 3.1 `CLAUDE.project.md` archive

`./CLAUDE.project.md`을 *지우는 게 아니라* 이름만 바꾼다 — 디자이너가 다시 켜고 싶을 때 작업 흔적을 잃지 않게.

```
mv ./CLAUDE.project.md ./CLAUDE.project.md.kd-archived-<YYYYMMDD-HHmm>
```

응답에 archived 경로 1줄 노출 — "이 프로젝트에서 끄기 전 상태는 `CLAUDE.project.md.kd-archived-YYYYMMDD-HHmm`로 보관해뒀어요".

#### 3.2 프로젝트 루트 `./CLAUDE.md` 정리 (대칭: §4 전역 끄기와 동일 패턴)

`/kd:디자인초기설정` §2.2가 박은 import 라인을 정리한다. 처리 절차:

1. **`./CLAUDE.md` 존재 여부 확인** — `test -f ./CLAUDE.md`. 없으면 스킵(이미 정리됨).

2. **3-way 분기** — 다음 우선순위로 한 가지만 실행:

   **3.2-a. `./.kd-no-prior-claude-md` 마커 발견** (우리 플러그인이 새로 만든 파일):
   - `rm ./CLAUDE.md` — 파일 자체 제거.
   - `rm ./.kd-no-prior-claude-md` — 마커도 정리.

   **3.2-b. `./CLAUDE.md.kd-backup-*` 백업 발견**(가장 최근 1개, timestamp 큰 것):
   - `cp <백업> ./CLAUDE.md` — *원본 그대로* 복원. 사용자가 손댔던 다른 내용까지 그대로.
   - 백업 파일 자체는 *지우지 않음*(보관) — 비파괴 원칙.

   **3.2-c. 마커만 있고 백업도 마커도 없음** (비상 fallback — 백업 사라진 경우):
   - `Read ./CLAUDE.md`로 마커 쌍 개수 확인.
   - **마커 쌍이 정확히 1쌍일 때만** 실행: `<!-- kd:designer-mode:start -->`부터 `<!-- kd:designer-mode:end -->`까지(양 끝 마커 포함) 통째로 제거. 마커 밖 사용자 원본 텍스트는 보존.
   - 0쌍 또는 2쌍 이상이면 사용자에게 보고 후 *중단*("`./CLAUDE.md`에 디자이너 마커 상태가 평소와 달라요 — 누가 손댔을 가능성이 있어 자동 정리를 멈췄어요. 한 번 봐주세요.").

3. **검증** — 처리 후 `Read ./CLAUDE.md`(파일이 남아 있다면)로 다시 읽어 마커가 *0개*인지 확인. 남아 있으면 응답에 경고 + 백업 경로 안내.

### 4. 전역 끄기 — 원본 복원

선택지가 *이 컴퓨터 전체에서 끄기* 또는 *둘 다 끄기*면 진행.

#### 4.1 백업 파일 탐색
`ls ~/.claude/CLAUDE.md.kd-backup-*` 결과에서 *가장 최근*(timestamp 큰 것) 1개 선택.

#### 4.2 분기

| 백업 상태 | 동작 |
|---|---|
| 정상 백업 발견 | `cp <백업> ~/.claude/CLAUDE.md` 로 *원본 그대로* 복원 |
| `~/.claude/.kd-no-prior-claude-md` 마커 발견 (원래 없던 파일) | `rm ~/.claude/CLAUDE.md` + 마커도 정리 |
| 백업도 마커도 없음 (마커만 있고 백업 사라짐) | **마커 사이만 잘라내기** — 시작·끝 마커 포함 그 사이 전부 삭제, 마커 밖 사용자 원본 텍스트는 보존 |

#### 4.3 마커 사이 잘라내기 fallback (4.2 세 번째 분기)
백업이 사라진 비상시에만:

1. `Read` `~/.claude/CLAUDE.md`
2. `<!-- kd:designer-mode:start -->`부터 `<!-- kd:designer-mode:end -->`까지(양 끝 마커 포함) 통째로 제거
3. `Edit` `replace_all: false` — 마커 쌍이 *정확히 1개*일 때만 실행. 0개면 이미 정리됨, 2개 이상이면 사용자에게 보고 후 중단(누가 손댔을 가능성).

#### 4.4 검증
복원 후 `Read`로 다시 읽어 마커가 *0개*인지 확인. 남아 있으면 사용자에게 경고 + 백업 경로 안내(수동 정리 권유).

### 5. 결과 안내

성공 응답 패턴 (둘 다 끄기):
> 디자이너 모드 **꺼졌어요**.
>
> - 이 프로젝트: 작업 기록은 `CLAUDE.project.md.kd-archived-...`에 보관 (다시 켤 때 복구 가능). 프로젝트의 `CLAUDE.md`도 원래대로 돌려놨어요.
> - 이 컴퓨터 전체: `~/.claude/CLAUDE.md` 원본 그대로 돌아갔어요
>
> 다시 켜고 싶으면 언제든 `/kd:디자인초기설정` 한 번이면 돼요.

이 프로젝트만 끈 경우 두 번째 줄 생략, 전역만 끈 경우 첫 줄 생략. "프로젝트의 `CLAUDE.md`도 원래대로 돌려놨어요" 1줄은 §3.2가 실제로 동작했을 때만(파일이 있어 처리한 경우) 노출.

### 6. "이미 꺼져 있음" 분기
§1 진단에서 둘 다 안 켜져 있으면:

> 이미 꺼져 있어요 — 끌 게 없어요.
>
> 다시 시작하고 싶으면 `/kd:디자인초기설정`으로 켤 수 있어요.

## 안전 장치

- **명시 동의 (전역 복원만 해당)** — `~/.claude/CLAUDE.md` 수정은 `AskUserQuestion`으로 동의받음. "이 프로젝트만 끄기" 옵션을 항상 기본·안전 위치로(CLAUDE.md §5).
- **archive, not delete** — 프로젝트 `CLAUDE.project.md`은 *지우지 않고 이름만 변경*. 디자이너 작업 흔적 보존.
- **백업 우선 복원 (전역·프로젝트 루트 동일 정책)** — `~/.claude/CLAUDE.md`·`./CLAUDE.md` 둘 다 *반드시 백업 파일에서* 복원. 백업 없을 때만 마커 사이만 잘라내는 보수적 fallback. 우리 플러그인이 *새로 만든* 파일임을 `.kd-no-prior-claude-md` 마커로 판단해 통째 제거하는 분기도 동일.
- **마커 검증** — 복원 후 마커가 0개인지 확인. 1개라도 남아 있으면 경고.
- **외부 도구 호출 금지** — `git`/`npm`/네트워크 X. 이 명령은 *모드 끄기*만.
