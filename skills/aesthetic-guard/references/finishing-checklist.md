# Finishing Checklist — 출력 직전 마감 표준

> `aesthetic-guard` SKILL 본체 §출력 직전 자체 점검 8개 *외*에 통과시켜야 할 마감 표준. 본체 8개는 *반드시*, 이 문서는 *추가 점검*.
>
> 이 문서는 `aesthetic-guard`가 코드 출력 직전 *반드시 Read*. 단순 카탈로그가 아니라 점검 항목이라 lazy load 강제력이 핵심.

---

## 1. 차원별 가드

| 차원 | 가드 |
|---|---|
| **타이포** | display + body 두 결 페어링. 한국어 본문은 Pretendard / Spoqa Han Sans / SUIT 같은 한글 우선 패밀리 (디자이너 프로필에 명시 폰트 있으면 그쪽). 줄높이·자간은 디스플레이/본문 다르게. **한국어 본문엔 `break-keep`(Tailwind, = `word-break: keep-all`) 강제** — 한글 단어 중간 줄바꿈 방지. |
| **색** | 지배색 1-2 + 강조색 1. 균등 분포 X. CSS 변수로 일관성. 디자이너 프로필 회피 색은 절대 X. **새 색은 발명하지 말고 OKLCH로 합성** — 기존 팔레트 확장 필요 시 `oklch(L C H)`로 *지각 명도(L) 균등하게* 조정해 같은 결로. RGB·HSL 직접 합성은 지각 비균질해 *어울리지 않는 색*이 자주 발생. |
| **공간** | 의도적 여백 OR 의도적 밀도 — 어중간 X. 비대칭/오버랩/그리드 깨기 적극 검토. |
| **모션** | 한 번 잘 짜인 페이지 진입(staggered reveal)이 산발 마이크로 인터랙션보다 강함. CSS-only 우선, React면 Motion(framer-motion). |
| **배경·디테일** | 단색 배경 X. 그라디언트 메쉬·노이즈·미세 패턴·레이어 투명도·드라마틱 그림자 중 *프로젝트 톤에 맞는 1-2개*. |

## 2. 미학 결정 — 추가 점검

- [ ] 흔한 폰트 단독으로 박지 않았는가 (Inter/Roboto/system-ui/Open Sans/Lato — 디자이너 프로필 명시 폰트 우선, 없으면 의도적 display + 다른 결의 body 페어링)
- [ ] 카드 그리드가 모든 정보를 같은 무게로 깔지 않았는가
- [ ] 디자이너 프로필 §피하고 싶은 디자인 + 프로젝트 §피하고 싶은 디자인 슬롯 어휘가 들어가지 않았는가
- [ ] 미니멀/맥시멀 강도가 코드 자체에도 반영됐는가 (어중간 X)
- [ ] 한국 서비스면 한국어 실제 카피·한국 이름인가
- [ ] 한국어 슬롭 형용사("혁신적인"·"차세대"·"원활한"·"게임 체인저") 안 들어갔는가 (자세한 어휘 카탈로그는 `ai-slop-vocab.md` §3)
- [ ] *기억에 남길 한 가지*가 화면에서 두드러지는가

---

## 3. 마감 디테일 — 자주 빼먹는 자리

- [ ] 정렬·간격 픽셀 단위 정확성 — 토큰(`p-4`/`gap-6`) 안 쓰고 임의 값(`p-[17px]`) 박혔으면 토큰으로
- [ ] 입력 컴포넌트 라벨/에러/도움말 자리 — `<label htmlFor>` 연결, `<input>` `name`/`id` 일치
- [ ] focus-visible 링/아웃라인 (키보드 접근성). focus-visible 없으면 `outline-none` 단독 X (본체 8개의 cursor-pointer+hover+focus-visible과 별개로 *모든* 인터랙티브 요소 점검)
- [ ] 정확히 정렬된 대칭만 → 의도적 비대칭/오버랩이 더 살아있음
- [ ] 같은 디자이너의 여러 화면이 *같은 폰트·같은 색조합·같은 레이아웃*으로 수렴하지 X (세대 간 수렴 회피)

---

## 4. 강도 매칭

미니멀 방향 = 코드도 절제(여백·정밀 타이포·CSS 변수 정리). 맥시멀 방향 = 코드도 풍부(레이어·애니메이션·다중 배경). *방향과 코드 강도가 맞아야* 의도가 산다.

- [ ] 시작 단계에서 정한 *한 방향*과 코드 강도가 일치하는가
- [ ] 어중간한 평균에 머무르지 않았는가
