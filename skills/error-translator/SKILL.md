## 목적
디자이너에게 에러 부담을 떠넘기지 않는다. 자동 회복이 먼저, 친화 메시지는 회복 실패 시에만.

## 발동 조건

### 발동
- 다른 Skill·도구 실행 후 stderr 또는 비정상 종료 코드 감지 (자동 가로채기)
- 사용자 자연어로 에러를 언급한 모든 경우

### 발동 X
- 정상 종료 + stderr 없음
- 단순 경고 (WARN/deprecation 류) — 무시하거나 한 줄로 묶어 안내
- 사용자가 명시적으로 "raw error 보여줘" 요청 (페르소나 §발동 X와 일치)

## 처리 흐름

### 1. 에러 분류 (메인 모델)
빠른 패턴 매칭으로 회복 시나리오를 정한다:

| 패턴 | 분류 | 자동 회복 |
|---|---|---|
| `Cannot find module`, `MODULE_NOT_FOUND`, `ENOENT package.json` | 의존성 누락 | 패키지 매니저로 install |
| `EADDRINUSE`, `port .* already in use` | 포트 충돌 | 다음 비어있는 포트로 재시도 |
| `ETIMEDOUT`, `ECONNRESET`, `network` | 일시 네트워크 | 1회 재시도 |
| `EACCES`, `permission denied` | 권한 | 자동 회복 X — 친화 메시지로 즉시 전환 |
| 컴파일/문법 (`SyntaxError`, `TS2304` 등) | 코드 오류 | 자동 회복 X — 친화 메시지로 전환 |
| 기타 | 미분류 | 친화 메시지로 전환 |

### 2. 자동 회복 시도 — Haiku Subagent 위임
정형 회복은 메인 컨텍스트를 채우지 않도록 Haiku Subagent에 위임 (PRD §12).

위임 프롬프트 (Subagent는 본 대화 컨텍스트 모름):
> 다음 에러를 회복하라. 회복 시나리오는 [의존성 설치 / 포트 변경 / 재시도] 중 [X]다.
>
> 에러 원문:
> ```
> <stderr 텍스트>
> ```
>
> - 작업 디렉토리: `<cwd>`
> - 패키지 매니저: `<lockfile로 추론, 없으면 npm>`
> - 원래 실행하려던 명령: `<원 명령>`
>
> 회복 명령 실행 → 원 명령 재시도 → 통과 여부까지 확인. 결과는 `status: pass|fail` + `actions: [...]` 정형 보고. 영어 출력 그대로 사용자 노출 X.

### 3. 회복 성공 응답
한국어 1~2줄로 *무엇을 처리했는지*만 알린다.

> **필요한 도구가 빠져 있어서 자동으로 설치**(`npm install`)했어요. 이제 다시 실행됐어요.

### 4. 회복 실패 응답 — 두 블록 패턴
페르소나 §5와 일치. **무엇이 안 됐는지** + **개발자에게 보여주실 메시지**.

> **저장이 안 됐어요** — 코드 일부에 문법 오류가 있어 보여요(`SyntaxError`). 이건 제가 자동으로 못 고쳐서 도움이 필요해요.
>
> 📋 **개발자에게 복사해 보내주세요:**
> ```
> file: components/Button.tsx:42
> SyntaxError: Unexpected token '<'
>   at parse (/.../parser.js:120:5)
> ```

이 두 블록은 *원본 stderr를 코드 블록 안에 가둬* 노출한다. 본문(친화 메시지)에는 영어가 단독으로 떠 있지 않게.

## Subagent 위임
- **정형 회복 실행** (의존성 설치, 포트 재시도, 단순 재시도) → **Haiku Subagent** (`Task` tool)
- 메인 모델은 (a) 분류 (b) 사용자 톤 응답만 담당. 명령 실행은 Haiku.
- 위임 프롬프트는 §2 참조. Subagent는 본 대화 컨텍스트 모름 가정 → 에러 원문·작업 디렉토리·패키지 매니저·원 명령까지 전부 명시.
- PRD §12 라우팅 표 일치 (메인 가로채기 + Haiku 정형 회복).

## 응답 톤
- 한국어, 영어 스택트레이스 *본문에 단독 노출 X* (코드 블록 안에서만)
- 회복 성공 → 짧게, 회복 실패 → 두 블록 패턴
- 응답 끝 다음 행동 1개 제안 (`designer-persona` 글로벌 원칙)
  - 성공: "이제 이어서 화면 보여드릴까요?"
  - 실패: "이 메시지 그대로 개발자에게 보내드릴까요?"
