# aesthetic-guard

## 목적
디자이너가 디자이너 모드를 켰을 때 *평범한 AI 화면*이 나오는 걸 막는다. 평균에 머무는 디자인보다 *한 방향에 과감하게 commit한* 디자인이 항상 낫다 — 미니멀이면 정밀하게, 맥시멀이면 풍부하게, 어중간 X. 프로덕션 수준의 *마감 디테일*과 *창의적 선택*을 주저하지 X. 응답 톤은 `designer-persona`가 잡고, 이 Skill은 *코드 출력의 미학 품질*만 본다.

## 시작 전 — 한 방향에 commit

코드 출력 *직전* 세 단계로 방향을 정한다:

1. **글로벌 디자이너 프로필** — `~/.claude/CLAUDE.md` §디자이너 프로필 (레퍼런스 + 본인 대표 작업 URL + 누적된 회피 + 평소 톤). 본인 대표 작업 URL은 *디자이너 현재 결*의 가장 강한 신호 — 톤·색·타이포·공간감 다차원 추출.
2. **프로젝트 미학 컨텍스트** — `CLAUDE.project.md` 안 디자인 철학·톤·레퍼런스·피하고 싶은 디자인·토큰
3. **기존 코드에서 유추** — `globals.css`/`tailwind.config.*`/`components/` 안 색·폰트·둥글기·간격 결정. 디자인 시스템이 이미 있으면 *거기서 톤·결을 추출*해 §철학·§톤 후보로.

위 3단계로도 *비어있고 코드 추출도 안 되는 차원*만 §"부족한 차원 묻기"로 — 명시적으로 디자이너에게 묻는다(`AskUserQuestion` 1개). 묻지 않고 평균에 머무르지 X.

**방향 결정의 핵심**: 어중간한 평균 X. 미니멀이면 *제대로* 미니멀(여백·정밀 타이포), 맥시멀이면 *제대로* 맥시멀(정교한 모션·레이어). 톤 후보는 좁게 잡지 X — 미니멀 / 맥시멀리스트 / 에디토리얼·매거진 / 브루탈·거친 / 레트로·퓨처리스틱 / 유기체·자연 / 럭셔리·정제 / 플레이풀·장난기 / 아르데코·기하 / 파스텔·소프트 / 산업적·실용 등 *11개 이상 결*에서 고른다.

**기억에 남길 한 가지**: 방향을 정한 뒤, *이 화면을 본 사람이 기억할 한 가지*를 한 단어로 정한다(과감한 비대칭 / 한 글자 크게 / 의도적 빈 공간 / 손글씨 헤딩 등). 그 한 가지가 화면 전체에서 두드러지게.

**세대 간 수렴 회피**: 같은 디자이너가 여러 화면을 만들 때 *같은 폰트·같은 색조합·같은 레이아웃*으로 수렴하지 X. 프로젝트 결과 화면 의도가 다르면 다르게.

## AI-slop 회피 리스트

다음은 *명시적으로 피한다*. 디자이너 프로필이 "이 클리셰는 OK"라고 명시한 경우만 예외.

**타이포**
- 흔한 폰트 단독: Inter, Roboto, Arial, Helvetica, system-ui, Open Sans, Lato — 디자이너 프로필 명시 폰트 우선, 없으면 의도적 display + 다른 결의 body 페어링
- 무지개 그라디언트 텍스트 (특히 보라→파랑 / 분홍→보라)

**색**
- 보라(`#6366f1`/`#7c3aed` 류) 그라디언트 + 흰 배경
- 모든 카드가 똑같이 옅은 회색 그림자
- 강조 없이 균등 분포된 8-10가지 색

**레이아웃**
- 카드 그리드 — 같은 크기 박스가 정렬된 채 모든 정보가 같은 무게
- 화면 맨 위 큰 글씨 + 부제 + CTA 버튼 한 줄, 같은 구도의 반복
- 정확히 정렬된 대칭 — 의도적 비대칭/오버랩이 더 살아있음

**아바타·인물·로고 — 이니셜 박스/원형은 가장 흔한 AI slop 신호**
- **이니셜 박스/원형 아바타** ("Kim" → 회색 원에 "K", "박지우" → 검정 사각형에 "박")
- **이니셜 박스/원형 브랜드 로고** (검정 사각형 + 흰 'S', 색 원 + 'K' 같은 *최소 결정만 한 로고*)
- 회색 사람 실루엣 아이콘 / 단색 원 + 무작위 색

→ 발견 즉시 **§"이니셜 박스 회피 — 적극 분기"**로 처리(§5 위).

**더미 데이터**
- "Lorem ipsum"
- "John Doe", "user@example.com", "Sample Title"
- 의미 없는 숫자 (`123`, `999`, `0`) — 한국 서비스면 한국어 실명·실제 카피가 화면을 살림

## 차원별 가드

| 차원 | 가드 |
|---|---|
| **타이포** | display + body 두 결 페어링. 한국어 본문은 Pretendard / Spoqa Han Sans / SUIT 같은 한글 우선 패밀리 (디자이너 프로필에 명시 폰트 있으면 그쪽). 줄높이·자간은 디스플레이/본문 다르게. |
| **색** | 지배색 1-2 + 강조색 1. 균등 분포 X. CSS 변수로 일관성. 디자이너 프로필 회피 색은 절대 X. |
| **공간** | 의도적 여백 OR 의도적 밀도 — 어중간 X. 비대칭/오버랩/그리드 깨기 적극 검토. |
| **모션** | 한 번 잘 짜인 페이지 진입(staggered reveal)이 산발 마이크로 인터랙션보다 강함. CSS-only 우선, React면 Motion(framer-motion). |
| **배경·디테일** | 단색 배경 X. 그라디언트 메쉬·노이즈·미세 패턴·레이어 투명도·드라마틱 그림자 중 *프로젝트 톤에 맞는 1-2개*. |

## 이니셜 박스 회피 — 적극 분기 (브랜드 로고·아바타 공통)

문자 박스/원형은 *가장 흔한 AI slop 신호*. 그냥 박지 X — 박기 *전에* 행동 카탈로그를 거친다.

### 박지 않는 길 (순서대로)

1. **있는 자산 먼저** — `public/`·`asset/`·`assets/`에 `logo.*`/`brand.*`/`icon.*`/`*.svg` 같은 파일이 있는지 `Bash ls` 또는 `Glob`으로 빠르게 확인. 있으면 그걸 사용. 디자이너가 *이미 가진 자산을 모르는 상태*에서 박지 X.

2. **디자이너에게 의견 묻기** (`AskUserQuestion`) — 객관식:
   > "**브랜드 로고/아이콘 자리**예요. 어떻게 채울까요?"
   > - "**로고 파일 갖고 있어요**" → "어디에 넣어드리면 될지(`public/logo.svg` 같은 경로) 알려드릴게요" 안내 후 *디자이너가 파일 넣을 때까지 일단 placeholder 처리* (placeholder는 ④ 분기)
   > - "**원하는 결의 아이콘**이 있어요" → 키워드 받기 → lucide-react·심볼·일러스트 후보 1-3개 보여드림. 디자이너가 고르면 박음.
   > - "**제가 추천해드려요**" → 서비스 결(§톤·§철학·§레퍼런스)에 맞는 후보 2-3개 코드에 박은 채 보여드리고 디자이너가 고름. 디자이너 첨부 레퍼런스가 있으면 거기서 결을 가져옴.
   > - "잘 모르겠어요" → ③ 추천 분기와 동일

3. **불가피하게 placeholder가 필요한 경우** — 이니셜 박스/원형 *대신*:
   - 서비스 결에 맞는 lucide-react 단순 심볼 (카페면 `Coffee`, AI 도구면 `Sparkles`, 도서면 `BookOpen` 등) + 톤 색
   - 또는 *디자인 결정이 들어간 모노그램 타이포* (`text-2xl font-serif tracking-tight italic` 같은 결 있는 글자) — 단순 sans-serif bold 박스 X
   - 어느 쪽이든 *임시*임을 디자이너에게 명시: "지금은 임시로 `Sparkles` 심볼을 박았어요. **진짜 로고**가 있으면 `public/logo.svg`로 넣어주시면 자동 교체돼요."

4. **사용자 아바타도 동일** — 이니셜 박스/원형 X. `https://i.pravatar.cc/150?img={1-70}`(실제 인물 사진) 또는 DiceBear 일러스트(`lorelei`/`notionists`/`shapes`) 우선. 디자이너 결에 맞는 스타일로.

→ 이 분기는 *과적합 X*. 디자이너가 "그냥 박아주세요"라고 명시 동의하면 즉시 박는다(③ placeholder 분기로). 단, 결과 응답에 "임시로 박았어요" 한 줄.

## Asset 구축 — 콘텐츠가 화면을 살린다

회색 placeholder·이니셜 아바타·"Lorem ipsum"이 화면을 죽인다. 진짜 콘텐츠를 박는다.

### 이미지·사진
- **결정론적 URL** 우선 — 매번 다른 이미지가 떨어지지 않게:
  - `https://picsum.photos/seed/{고유키}/800/600` — Lorem Picsum, seed 같으면 이미지 고정
  - `https://source.unsplash.com/{사이즈}/?{키워드}` — 카테고리 기반 (예: `?cafe,korea`)
- 디자이너에게 *후보 1-3장*을 코드에 박은 채 "이 중 어떤 결이 맞나요?" 확인. 마음에 드는 게 있으면 `asset/`에 다운로드(`curl`)해서 로컬화.
- 프로젝트 결이 정해져 있으면 디자이너 확인 없이 그 결대로 한 번에 박음.

### 사람 얼굴·아바타
- **이니셜 박스 절대 X**.
- 무료 옵션:
  - `https://i.pravatar.cc/150?img={1-70}` — 실제 인물 사진
  - DiceBear (`https://api.dicebear.com/7.x/{스타일}/svg?seed={이름}`) — 일러스트 아바타. 스타일은 프로젝트 결에 맞춰: `lorelei`(따뜻한 일러스트), `notionists`(노션 결), `shapes`(추상), `fun-emoji`(플레이풀)
- 한국 서비스 더미는 *한국 이름*: 박지우·김민서·이서연 같은 평범한 실명 풀.

### 아이콘
- 기본 lucide-react. 2D 흑백 라인 일관성. 다른 셋(Phosphor·Tabler·Heroicons) 섞지 X.
- 프로젝트 결이 럭셔리/에디토리얼이면 Phosphor(thin·duotone)도 검토.

### 더미 텍스트
- 한국 서비스면 한국어 실제 카피로. *서비스 맥락에 맞는 진짜 문장*. "다정한 카페 안내" 서비스면 "오후 2시, 햇살 잘 드는 창가 자리" 식.
- 숫자도 의미 있게 — 가격은 한국 시세, 거리는 m/km 단위.

### 외부 사례 참조 (필요할 때만)
디자이너가 레퍼런스 URL을 줬는데 톤·결을 확신할 수 없으면 `WebFetch`로 한 번 분석해 *카피·구도·색감*만 추출. 단, 페이지 분석은 토큰 비용이 있어 *디자이너 명시 레퍼런스가 있을 때만*.

## 강도 매칭

미니멀 방향 = 코드도 절제(여백·정밀 타이포·CSS 변수 정리). 맥시멀 방향 = 코드도 풍부(레이어·애니메이션·다중 배경). *방향과 코드 강도가 맞아야* 의도가 산다.

## 부족한 차원 묻기 — *코드 추출 실패 시* 명시적 1개

이 플러그인 페르소나는 Claude Code 기능을 잘 모르는 디자이너. 미학 결정을 *암묵적으로 평균에 맞추면* AI slop 직행 — 그래서 부족 차원은 *적극적으로 묻는다*. 단, 한 번에 1개만, 자연스럽게.

**묻기 전에 — 코드 추출 우선** (불필요한 질문 차단):
1. `globals.css`/`tailwind.config.*` → 색·폰트·둥글기·간격 추출
2. `components/` → 기존 스타일 결 추출 (radius·shadow·typography 어떤 결?)
3. `~/.claude/CLAUDE.md` §디자이너 프로필 (전역 켜져 있으면)
4. `CLAUDE.project.md` §디자인 시스템 슬롯들

위 4단계로도 *비어있는* 차원만 디자이너에게 묻는다.

**좋은 질문 패턴** (디자이너 어휘):
- 톤이 비어있을 때: "이 화면 *어떤 분위기*로 가져가시고 싶어요? 한 단어라도 좋아요. (예: 미니멀 / 에디토리얼 / 플레이풀 / 럭셔리 / ...)"
- 레퍼런스가 비어있을 때: "참고할 만한 화면이 1개 있으면 보여주세요. URL·이미지·서비스 이름 다 좋아요."
- 철학이 비어있고 톤만으론 부족할 때: "이 화면이 *제일 우선해야 할 한 가지*가 뭘까요? 정보 빠르게 / 분위기 / 신뢰감 / 재미 중에서요."

**안 좋은 질문 패턴 (회피)**
- "이 프로젝트의 디자인 철학은?" — 너무 열림
- "색·간격·둥글기·타이포 다 정해주세요" — 한 번에 너무 많은 결정
- "어떤 폰트 쓸까요?" — 디자이너 어휘 X (어휘는 *결*·*분위기*·*톤*)
- "피하고 싶은 디자인 있으세요?" — *사전 인터뷰* X. 회피는 §회피 능동 유도로 사후 처리.

**판단 가이드**
- 글로벌 프로필이 채워져 있으면 그대로 가져다 쓰고 *묻지 X*. 디자이너가 "이번엔 다른 톤" 명시할 때만 재질문.
- `import-existing`이 추출한 토큰·폰트가 있으면 그게 답의 일부. 거기서 *부족한 차원*만.
- 결정 피로 글로벌 §6 — 한 사이클 질문 3회 상한. 다른 Skill이 이미 묻고 있으면 침묵.
- 모든 객관식엔 "**잘 모르겠어요 / 추천해주세요**" 옵션 필수 (글로벌 §6).

## 회피 사후 누적 — 능동 유도 + 부정 신호 인식

회피 사전 인터뷰는 X — 디자이너에게 *AI 클리셰 카탈로그*를 떠넘기는 부담. 대신 두 경로로 누적.

### A. 능동 유도 (작업 마무리 시 가볍게 한 번)

화면 산출물을 디자이너에게 보여준 직후, *부정 신호가 없어도* 한 번 가볍게 묻기:

> 이 화면에서 *마음에 안 드는 부분* 있으면 알려주세요. 누적해뒀다가 다음 화면부터 빼요. 없으면 "괜찮아요"만 답해주세요.

- 한 사이클당 1회만 (글로벌 §6 질문 3회 상한 안). 강제·재촉 X.
- "괜찮아요"면 바로 다음 행동으로 — 더 묻지 X.
- 답이 들어오면 §B처럼 누적.

### B. 부정 신호 인식 (자연어)

- "너무 흔해 보여" / "어디서 본 듯한" / "스타트업 같은 룩"
- "이 색 좀…" / "폰트 별로" / "이거 빼고"
- "Linear/Notion/Vercel처럼 X말고" 같은 *부정 비교*

### 누적 처리

거부된 차원 식별 → 디자이너 발화 *그대로* 슬롯에 한 줄 추가 → 다음 출력에서 회피.

**누적 위치 결정**:
- 첫 등장 → `CLAUDE.project.md` §피하고 싶은 디자인 (이 프로젝트 한정).
- 같은 결의 회피가 *2회+ 반복*되면(다른 화면·다른 답이라도 같은 어휘 결) → 전역 승격 권유:
  > "지난 화면들에서 *비슷한 결*을 계속 빼셨네요({{디자이너 어휘}}). **이 컴퓨터의 모든 프로젝트에서 같이 빼둘까요?** 그러면 다른 프로젝트 가서도 자동으로 회피해요."
  > - "네, 전역에 추가" → `~/.claude/CLAUDE.md` §피하고 싶은 디자인 슬롯에 추가 (전역이 켜져 있을 때만; 안 켜져 있으면 "전역 모드를 켜시면 가능해요" 안내 1줄)
  > - "이 프로젝트에만" → 프로젝트 슬롯 유지
  > - "잘 모르겠어요" → 이 프로젝트에만 (기본값)
- 전역으로 승격된 회피는 다음부터 *모든 프로젝트*에서 자동 적용 — 매번 같은 결을 빼달라고 반복 요청할 필요 X.

**과적합 X**: 디자이너가 한 번 부정 반응했다고 *영원히* 그 어휘를 회피하지 X — 슬롯에 박는 건 *다음 출력 회피*용 단서이고, 디자이너가 "다음엔 그 결로" 명시하면 무시하고 그 결로.

**마감 디테일** (자주 빼먹는 자리)
- [ ] 클릭 가능한 요소(`<button>`/`<a>`/`onClick`/`role="button"`/`role="link"`)에 `cursor-pointer` 명시. shadcn/ui Button은 기본이지만 *`<div onClick>`* 같은 커스텀 클릭 영역은 자주 빠짐

- [ ] focus-visible 링/아웃라인 (키보드 접근성). focus-visible 없으면 `outline-none` 단독 X
- [ ] disabled 상태 시각 대비 + `cursor-not-allowed` + `aria-disabled` (실제로 동작 막아야 하면 `disabled` 속성도)

- [ ] 정렬·간격 픽셀 단위 정확성 — 토큰(`p-4`/`gap-6`) 안 쓰고 임의 값(`p-[17px]`) 박혔으면 토큰으로
