## 목적
기존 코드 훼손 위험 최소화하며 디자이너가 안전하게 끼어들기. 컴포넌트 인덱스 단일 참조점 생성.

## 발동 조건
- 발동: 기존 프로젝트 가져오기 자연어
- 발동 X: 빈 디렉토리 (그땐 `new-service`)

## 처리 흐름
> ⚠️ Phase 5 placeholder. 본 로직은 Phase 5에서 작성.
> 핵심:
> 1. 디자인 시스템 존재 여부 확인 (있으면 추출, 없으면 권유 메시지)
> 2. 컴포넌트 hook·props 의존성 더미 환경에서 이해 가능하게 자동 문서화
> 3. `CLAUDE.project.md` `## 사용 가능한 컴포넌트` 섹션 첫 생성 (이름·경로·핵심 props·variant)
> 4. 디자인 시스템 *없는 프로젝트*는 만들기 권유 (만드는 방식은 O-5 추적, 1차에선 권유 메시지만)

## Subagent 위임
- 본 Skill 자체가 Sonnet Subagent (코드 스캔 + 컴포넌트 문서화)

## 응답 톤

- 외부 도구: Glob/Grep (코드 스캔), Read (컴포넌트 분석), Write (`CLAUDE.project.md` 생성)
