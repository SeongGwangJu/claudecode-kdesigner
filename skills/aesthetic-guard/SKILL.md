# aesthetic-guard

## 목적
디자이너가 디자이너 모드를 켰을 때 *평범한 AI 화면*이 나오는 걸 막는다. 한 방향 미학에 commit하고, 흔한 클리셰를 회피하고, 콘텐츠가 진짜 살아있게 한다. 응답 톤은 `designer-persona`가 잡고, 이 Skill은 *코드 출력의 미학 품질*만 본다.

## 시작 전 — 한 방향에 commit

코드 출력 *직전* 두 컨텍스트를 읽고 한 방향을 정한다:

1. **글로벌 디자이너 프로필** — `~/.claude/CLAUDE.md` §디자이너 프로필 (레퍼런스 + 본인 대표 작업 URL + 누적된 회피 + 평소 톤). 본인 대표 작업 URL은 *디자이너 현재 결*의 가장 강한 신호 — 톤·색·타이포·공간감 다차원 추출.
2. **프로젝트 미학 컨텍스트** — `CLAUDE.project.md` 안 디자인 철학·톤·레퍼런스·피하고 싶은 표현·토큰

방향 결정의 핵심: **어중간한 평균 X**. 미니멀이면 *제대로* 미니멀(여백·정밀 타이포), 맥시멀이면 *제대로* 맥시멀(정교한 모션·레이어). 두 컨텍스트가 *모두 비어 있고* 작업에 미학 결정이 필요하면 → §6 "부족한 차원 묻기"로.

## AI-slop 회피 리스트

다음은 *명시적으로 피한다*. 디자이너 프로필이 "이 클리셰는 OK"라고 명시한 경우만 예외.

**타이포**
- 흔한 폰트 단독: Inter, Roboto, Arial, Helvetica, system-ui, Open Sans, Lato — 디자이너 프로필 명시 폰트 우선, 없으면 의도적 display + 다른 결의 body 페어링
- 무지개 그라디언트 텍스트 (특히 보라→파랑 / 분홍→보라)

**색**
- 보라(`#6366f1`/`#7c3aed` 류) 그라디언트 + 흰 배경
- 모든 카드가 똑같이 옅은 회색 그림자
- 강조 없이 균등 분포된 8-10가지 색

**레이아웃**
- 카드 그리드 — 같은 크기 박스가 정렬된 채 모든 정보가 같은 무게
- 화면 맨 위 큰 글씨 + 부제 + CTA 버튼 한 줄, 같은 구도의 반복
- 정확히 정렬된 대칭 — 의도적 비대칭/오버랩이 더 살아있음

**아바타·인물**
- **이니셜 박스 아바타** ("Kim" → "K", "박지우" → "박") — 빈 데이터 plug. 진짜 콘텐츠가 들어갈 자리는 §5 "Asset 구축"으로 채울 것
- 회색 사람 실루엣 아이콘
- 단색 원 + 무작위 색

**더미 데이터**
- "Lorem ipsum"
- "John Doe", "user@example.com", "Sample Title"
- 의미 없는 숫자 (`123`, `999`, `0`) — 한국 서비스면 한국어 실명·실제 카피가 화면을 살림

## 차원별 가드

| 차원 | 가드 |
|---|---|
| **타이포** | display + body 두 결 페어링. 한국어 본문은 Pretendard / Spoqa Han Sans / SUIT 같은 한글 우선 패밀리 (디자이너 프로필에 명시 폰트 있으면 그쪽). 줄높이·자간은 디스플레이/본문 다르게. |
| **색** | 지배색 1-2 + 강조색 1. 균등 분포 X. CSS 변수로 일관성. 디자이너 프로필 회피 색은 절대 X. |
| **공간** | 의도적 여백 OR 의도적 밀도 — 어중간 X. 비대칭/오버랩/그리드 깨기 적극 검토. |
| **모션** | 한 번 잘 짜인 페이지 진입(staggered reveal)이 산발 마이크로 인터랙션보다 강함. CSS-only 우선, React면 Motion(framer-motion). |
| **배경·디테일** | 단색 배경 X. 그라디언트 메쉬·노이즈·미세 패턴·레이어 투명도·드라마틱 그림자 중 *프로젝트 톤에 맞는 1-2개*. |

## Asset 구축 — 콘텐츠가 화면을 살린다

회색 placeholder·이니셜 아바타·"Lorem ipsum"이 화면을 죽인다. 진짜 콘텐츠를 박는다.

### 이미지·사진
- **결정론적 URL** 우선 — 매번 다른 이미지가 떨어지지 않게:
  - `https://picsum.photos/seed/{고유키}/800/600` — Lorem Picsum, seed 같으면 이미지 고정
  - `https://source.unsplash.com/{사이즈}/?{키워드}` — 카테고리 기반 (예: `?cafe,korea`)
- 디자이너에게 *후보 1-3장*을 코드에 박은 채 "이 중 어떤 결이 맞나요?" 확인. 마음에 드는 게 있으면 `asset/`에 다운로드(`curl`)해서 로컬화.
- 프로젝트 결이 정해져 있으면 디자이너 확인 없이 그 결대로 한 번에 박음.

### 사람 얼굴·아바타
- **이니셜 박스 절대 X**.
- 무료 옵션:
  - `https://i.pravatar.cc/150?img={1-70}` — 실제 인물 사진
  - DiceBear (`https://api.dicebear.com/7.x/{스타일}/svg?seed={이름}`) — 일러스트 아바타. 스타일은 프로젝트 결에 맞춰: `lorelei`(따뜻한 일러스트), `notionists`(노션 결), `shapes`(추상), `fun-emoji`(플레이풀)
- 한국 서비스 더미는 *한국 이름*: 박지우·김민서·이서연 같은 평범한 실명 풀.

### 아이콘
- 기본 lucide-react. 2D 흑백 라인 일관성. 다른 셋(Phosphor·Tabler·Heroicons) 섞지 X.
- 프로젝트 결이 럭셔리/에디토리얼이면 Phosphor(thin·duotone)도 검토.

### 더미 텍스트
- 한국 서비스면 한국어 실제 카피로. *서비스 맥락에 맞는 진짜 문장*. "다정한 카페 안내" 서비스면 "오후 2시, 햇살 잘 드는 창가 자리" 식.
- 숫자도 의미 있게 — 가격은 한국 시세, 거리는 m/km 단위.

### 외부 사례 참조 (필요할 때만)
디자이너가 레퍼런스 URL을 줬는데 톤·결을 확신할 수 없으면 `WebFetch`로 한 번 분석해 *카피·구도·색감*만 추출. 단, 페이지 분석은 토큰 비용이 있어 *디자이너 명시 레퍼런스가 있을 때만*.

## 강도 매칭

미니멀 방향 = 코드도 절제(여백·정밀 타이포·CSS 변수 정리). 맥시멀 방향 = 코드도 풍부(레이어·애니메이션·다중 배경). *방향과 코드 강도가 맞아야* 의도가 산다.

## 부족한 차원 묻기 — 규칙 아닌 판단

미학 컨텍스트가 비어 있고 작업에 *그 차원이 정말 필요할 때만* 디자이너에게 자연스레 1개 묻는다. 한 번에 여러 차원 묻지 X.

**좋은 질문 패턴**
- 톤이 비어있을 때: "이 화면 *어떤 분위기*로 가져가시고 싶어요? 한 단어라도 좋아요."
- 레퍼런스가 비어있을 때: "참고할 만한 화면 1개 보여주실 수 있을까요? URL이나 이미지 다 좋아요."
- 회피가 비어있을 때: "혹시 *피하고 싶은* 표현 있으세요? (예: 보라 그라디언트, 카드 그리드)"

**안 좋은 질문 패턴 (회피)**
- "이 프로젝트의 디자인 철학은?" — 너무 열림
- "색·간격·둥글기·타이포 다 정해주세요" — 한 번에 너무 많은 결정
- "어떤 폰트 쓸까요?" — 디자이너 어휘 X (어휘는 *결*·*분위기*·*톤*)

판단 가이드:
- 글로벌 프로필이 채워져 있으면 그대로 가져다 쓰고 *묻지 X*. 디자이너가 "이번엔 다른 톤" 명시할 때만 재질문.
- `import-existing`이 추출한 토큰·폰트가 있으면 그게 답의 일부. 거기서 *부족한 차원*만.
- 결정 피로 글로벌 §6 — 한 사이클 질문 3회 상한. 다른 Skill이 이미 묻고 있으면 침묵.

## 회피 사후 누적 (사전 질문 X)

회피는 사전 인터뷰에서 묻지 않는다 — 디자이너에게 *AI 클리셰 카탈로그*를 떠넘기는 부담이라. 대신 디자이너가 결과에 부정 반응 보일 때 *그 어휘 그대로* 누적.

신호 패턴 (자연어 인식):
- "너무 흔해 보여" / "어디서 본 듯한" / "스타트업 같은 룩"
- "이 색 좀…" / "폰트 별로" / "이거 빼고"
- "Linear/Notion/Vercel처럼 X말고" 같은 *부정 비교*

처리: 거부된 차원 식별 → 디자이너 발화 그대로 슬롯에 한 줄 추가 → 다음 출력에서 회피. *프로젝트 한정*이면 `CLAUDE.project.md` §피하고 싶은 표현에만, *반복되는 평생 결*이면 `~/.claude/CLAUDE.md` §피하고 싶은 표현으로 승격.

## 출력 직전 자체 점검

코드 보내기 직전 자가 체크:
- [ ] 흔한 폰트 단독으로 박지 않았는가 (Inter/Roboto/system-ui)
- [ ] 이니셜 박스 아바타·"Lorem ipsum"·"John Doe"가 남아있지 않은가
- [ ] 카드 그리드가 모든 정보를 같은 무게로 깔지 않았는가
- [ ] 디자이너 프로필 회피 표현이 들어가지 않았는가
- [ ] 미니멀/맥시멀 강도가 코드 자체에도 반영됐는가 (어중간 X)
- [ ] 한국 서비스면 한국어 실제 카피·한국 이름인가

## Subagent 위임
없음. 메인 모델 컨텍스트에서 작동(`model: inherit`). 미학 결정은 메인 흐름 맥락이 필요해서 위임 X. PRD §12 라우팅 표 일치.

## 의존
- 다른 Skill: `designer-persona`(톤 가드, 이 Skill은 미학만), `design-system-guard`(토큰 일관성), `import-existing`/`new-service`(미학 컨텍스트 시드)
- 컨텍스트: `~/.claude/CLAUDE.md` 디자이너 프로필 섹션, `CLAUDE.project.md` 디자인 철학·레퍼런스·회피
- 외부 도구: `WebFetch`(레퍼런스 분석, 디자이너 명시 시만), `Bash curl`(이미지 로컬 다운로드, 디자이너 확정 시), `AskUserQuestion`(부족 차원 묻기)
- 참조: PRD §5 aesthetic-guard, §6 AI-slop 회피 원칙
