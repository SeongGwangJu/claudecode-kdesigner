# feedback-curator

## 목적
디자이너의 회피 어휘를 *원문 그대로* 누적해 다음 출력에 반영하고, 같은 결이 반복되면 전역으로 승격 권유. `aesthetic-guard`에서 분리한 *피드백 수집·전파* 단일 책임 모듈.

## 발동 조건

### 발동
- 화면 산출물 직후 작업 마무리 — *부정 신호가 없어도* 능동 유도 1회
- 자연어 부정 신호 감지 — "너무 흔해" / "이 색 좀…" / "폰트 별로" / "X말고"
- `aesthetic-guard`로부터 *디자이너 부정 반응 발견* 위임

### 발동 X
- 디자이너가 명시적으로 "괜찮아요" / 긍정 반응만 있는 경우
- 사전 인터뷰성 회피 묻기 — *사후*에만 동작
- 같은 사이클에 이미 능동 유도 1회 발동했으면 추가 X

## 처리 흐름

### A. 능동 유도 (작업 마무리 시 가볍게 한 번)

화면 산출물을 디자이너에게 보여준 직후, *부정 신호가 없어도* 한 번 가볍게 묻기:

> 이 화면에서 *마음에 안 드는 부분* 있으면 알려주세요. 누적해뒀다가 다음 화면부터 빼요. 없으면 "괜찮아요"만 답해주세요.

- 한 사이클당 1회만 (글로벌 §6 질문 3회 상한 안). 강제·재촉 X.
- "괜찮아요"면 바로 다음 행동으로 — 더 묻지 X.
- 답이 들어오면 §C처럼 누적.

### B. 부정 신호 인식 (자연어)

다음 패턴 감지 시 즉시 발동:
- "너무 흔해 보여" / "어디서 본 듯한" / "스타트업 같은 룩"
- "이 색 좀…" / "폰트 별로" / "이거 빼고"
- "Linear/Notion/Vercel처럼 X말고" 같은 *부정 비교*

### C. 누적 처리

거부된 차원 식별 → 디자이너 발화 *그대로* 슬롯에 한 줄 추가 → 다음 출력에서 회피.

#### 누적 위치 결정

**첫 등장** → `CLAUDE.project.md` §피하고 싶은 디자인 (이 프로젝트 한정, 슬롯 ID `project-avoidance`).

**같은 결의 회피가 *2회+ 반복*되면** (다른 화면·다른 답이라도 같은 어휘 결) → 전역 승격 권유:

> "지난 화면들에서 *비슷한 결*을 계속 빼셨네요({{디자이너 어휘}}). **이 컴퓨터의 모든 프로젝트에서 같이 빼둘까요?** 그러면 다른 프로젝트 가서도 자동으로 회피해요."
> - "네, 전역에 추가" → `~/.claude/CLAUDE.md` §피하고 싶은 디자인 슬롯에 추가 (전역이 켜져 있을 때만; 안 켜져 있으면 "전역 모드를 켜시면 가능해요" 안내 1줄)
> - "이 프로젝트에만" → 프로젝트 슬롯 유지
> - "잘 모르겠어요" → 이 프로젝트에만 (기본값)

전역으로 승격된 회피는 다음부터 *모든 프로젝트*에서 자동 적용.

- 전역: `<!-- kd:slot:avoidance -->`

(자세한 schema는 `plugin/SCHEMA.md` §2 참조)

### D. 과적합 회피

디자이너가 한 번 부정 반응했다고 *영원히* 그 어휘를 회피하지 X — 슬롯에 박는 건 *다음 출력 회피*용 단서이고, 디자이너가 "다음엔 그 결로" 명시하면 무시하고 그 결로.

## 트리거 충돌 처리 (참고)

토큰 변경 의도 → `design-system-guard` / 미학 결정·구도 → `aesthetic-guard` / 부정 신호 누적·전역 승격 → 이 Skill. 전체 표는 `plugin/SCHEMA.md` §5.

## Subagent 위임
없음. 메인 모델 컨텍스트에서 작동(`model: inherit`). 누적·승격 결정은 메인 흐름 맥락이 필요해서 위임 X.

## 응답 톤
- 한국어, 비유 보조, 응답 끝 다음 행동 1개 제안 (`designer-persona` 글로벌 원칙)
- 능동 유도는 *부담 안 주는 톤*: "없으면 '괜찮아요'만"
- 누적 시 한 줄 노출: "이 결({{어휘}})은 다음부터 빼둘게요"
- 전역 승격은 *왜 묻는지* 한 줄 + 디자이너 어휘 인용

## 의존
- 다른 Skill: `aesthetic-guard` (위임 받기), `designer-persona` (톤), `design-system-guard` (트리거 충돌)
- 슬롯: `project-avoidance` (프로젝트), `avoidance` (전역) — 정의는 `plugin/SCHEMA.md` §2
- 외부 도구: `Read` / `Edit` (CLAUDE.project.md / ~/.claude/CLAUDE.md), `AskUserQuestion` (전역 승격 동의)
