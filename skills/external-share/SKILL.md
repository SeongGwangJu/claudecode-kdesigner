## 목적
디자이너가 작업물을 외부에 공유하려 할 때, *어떻게 공유할지의 trade-off*(보안·외부 의존·지속성)를 디자이너 언어로 번역해 안내하고 가장 적합한 도구 *1개*를 추천·실행. 회사 프로젝트 사고를 사전 게이트로 차단하고, 공유 시점·대상을 자동 기록해 인계와 자연스럽게 이어준다.

## 발동 조건

### 발동
- 위 description 자연어 예시 + 동의어
- *외부에 보여줄 의도*가 명확한 발화 — "보여주고 싶어"는 대상이 *외부(클라이언트·팀·상사)*일 때만 발동, 본인 미리보기는 `preview`

### 발동 X
- 단순 저장 ("저장해줘"/"임시저장") → `safe-save`
- 본인 미리보기 ("보여줘"/"화면 켜줘") → `preview`
- 개발자 인계 ("개발자한테 넘기기"/"인계 정리") → `export-handoff`
- 점검만 ("점검해줘"/"확인해줘") → `quality-check`

발동 모호 시 1회 묻기 ("외부에 공유하시려는 거 맞나요? 본인 화면만 보시려면 '보여줘'라고 말씀해주세요").

## 처리 흐름

### 1. 의도 분류 (AskUserQuestion 1회)

`AskUserQuestion`. CLAUDE.md §5 — "왜 묻는지" 1줄 + "잘 모르겠어요" 옵션.

> 어떤 식으로 공유하실 건가요? *왜 묻냐면, 누구에게·얼마나 보여줄지에 따라 안전하게 풀어내는 방식이 다르거든요.*
>
> - **잠깐만 보여줄 거예요** (5분~1시간, 통화하면서 같이 보기) — 내 컴퓨터 켜진 동안만 통하는 임시 통로
> - **며칠 동안 시안 돌릴 거예요** (팀·동료 리뷰 받기) — 외부에 정식으로 올림, 무료 플랜 가능
> - **계속 볼 수 있게 하고 싶어요** (클라이언트가 며칠~몇 주 봄) — 외부 정식 + 비밀번호 권장
> - **진짜 외부 공개해요** (인터넷 누구나 보게) — 보안 점검 강화 후 정식 공개
> - **잘 모르겠어요** → 기본값: 두 번째 (며칠 시안 돌리기)

선택을 `INTENT = A | B | C | D` 로 보관.

### 2. Trade-off 안내 (선택 직후 본문 노출)

선택지 *밖의* 옵션도 가볍게 보여줘 디자이너가 결정 검증할 수 있게:

```
A. 잠깐만 보여주기 (임시 통로)
   ✓ 1분 셋업, 회원가입 X
   ✗ 내 컴터 끄면 사라짐, URL이 매번 바뀔 수 있음
   ⚠ 인터넷에서 내 컴터로 통로 뚫는 방식 — 회사 보안 정책 있다면 IT팀 확인

B. 며칠 시안 돌리기 (무료 정식 배포)
   ✓ 항상 접근 가능, URL 안 바뀜
   ✗ 외부 서비스 회원가입 필요 (이메일+비번)
   ⚠ 무료 플랜은 *공개*가 기본 — 검색엔진엔 안 잡혀도 URL 알면 누구나 봄

C. 비밀번호 걸어 공유 (지속)
   ✓ B와 같지만 지정한 사람만 봄
   ✗ 일부 도구는 유료 플랜이 필요할 수 있음

D. 진짜 외부 공개
   ⚠ 회사 프로젝트면 *반드시* 사전 점검:
     - 더미 데이터만 들어있나?
     - 회사 로고/내부 정보 노출 안 되나?
     - 도메인 정책은 IT팀과 합의됐나?
```

선택을 바꾸고 싶으면 다시 물어볼 수 있다고 한 줄 안내.

### 4. 사전 게이트 3중

세 가지가 *모두 통과*해야 §5로 진입. 하나라도 실패면 차단·자동 회복 시도(가능한 경우).

#### 4.1 빌드 검증 — `auto-validate` 위임

`Task` tool로 `auto-validate` Subagent 호출. *build script 우선* 명시:

> 외부 공유 직전 검증입니다. `build` script가 있으면 그것까지 돌려주세요(dev 서버 죽어있을 때만). 빈 화면·500 에러로 외부에 나가면 안 됩니다.

분기:
- `status: pass` → 통과
- `status: skipped` (검증 환경 없음) → 진행하되 응답에 1줄 안내 ("이 프로젝트는 빌드 검증 환경이 없어 그대로 진행했어요")
- `status: fail` → `error-translator` 위임. 자동 회복 성공 시 재검증, 실패 시 두 블록 패턴 응답하고 *공유 중단*

#### 4.2 민감정보 스캔 (신규)

회사 프로젝트 사고 1순위 차단점. 다음 패턴을 `Glob`+`Grep`로 검출:

| 패턴 | 의미 | 처리 |
|---|---|---|
| `.env`, `.env.local`, `.env.*` | 환경 변수 파일 존재 | `.gitignore`에 포함됐는지 확인 — 빠져있으면 *차단* |
| `(sk\|ghp\|gho\|glpat\|xox[bp])[_-][A-Za-z0-9_-]{16,}` | 흔한 API key prefix (OpenAI `sk-...`/`sk-proj-...`/`sk-ant-...` · GitHub `ghp_...`/`gho_...` · GitLab `glpat-...` · Slack `xoxb-...`/`xoxp-...`). prefix 다음 하이픈·언더스코어 양쪽 허용, 본문에 추가 하이픈 허용 | 어느 파일이든 발견 시 *차단* |
| `process\.env\.[A-Z_]+` | 클라이언트 코드에 환경변수 직접 참조 | Next.js의 `NEXT_PUBLIC_*` 외 패턴이면 경고 (공개 빌드에 박힘) |
| `CLAUDE.project.md` §공유 정책의 회사명 키워드 | 사전 등록된 회사 내부명 | 발견 시 *차단* + 어느 파일·줄 |

검출 시:
- *차단* — 진행 멈추고 어느 파일 어느 줄인지 표시
- *자동 마스킹 X* — 디자이너에게 결정권 (응답에 "이 정보를 제거하거나 .gitignore에 넣고 다시 시도해주세요" 안내)
- 회사명 키워드가 미설정이면 §6 회사 프로젝트 가드에서 묻고 캐시

스캔 범위: `git ls-files` 또는 staged + tracked + untracked(gitignore 제외). `node_modules/`·`.next/`·`dist/`·`build/`는 제외.

#### 4.3 더미 데이터 정책 검증

`CLAUDE.project.md` §더미 데이터 규칙(L79-84 부근)과 실제 코드 매칭:

- `mock/` 폴더 안에 데이터 분리됐는지
- 컴포넌트 안에 *하드코딩된 데이터*가 있는지 (배열·객체 리터럴 큰 덩어리 검출)
- 검출 시 *경고만* — 차단 X (디자이너가 의도적으로 그렇게 했을 수도)
- 회사 프로젝트(§6 가드 결과 = 예)면 *경고 → 확인 질문*으로 격상

검출 결과는 §8 응답에서 함께 노출.

### 7. 도구 추천 + 실행

§3 STACK + §1 INTENT 조합으로 도구 *1개* 추천. *콕 찍어 두지 않음* — LLM 시점에 가장 적합한 무료·간편 도구를 추천. 후보(참고):

| STACK | INTENT A (잠깐) | INTENT B/C/D (정식) |
|---|---|---|
| Next.js / React+Vite / Vue / Astro | `cloudflared tunnel` (무료 영구 URL 가능) 또는 `ngrok` (계정 필요) | Vercel CLI / Netlify CLI / Cloudflare Pages |
| 정적 HTML | 동상 | Netlify Drop / Cloudflare Pages / GitHub Pages |
| Expo | Expo Go QR (도구 자체 생성) | EAS Update |
| Flutter / 기타 | "이 스택은 추천 자동 매칭이 약해요. 옵션 정리해드릴게요" → 후보 2~3개 + trade-off → 사용자 선택 후 명시 동의 → 진행 |

추천 시 *왜 이 도구인지* 1줄 (예: "Next.js는 *Vercel CLI*가 가장 빠른데, 웹 한 줄 명령으로 끝나요").

도구 *콕 찍지 X* 정책 (PRD §5):
- 코드에 도구명 하드코딩 X (이 표는 *참고용 가이드*, LLM이 그 시점 판단)
- *디자이너 발화*에 도구명이 있으면 (예: "Vercel로 올려줘") 그 도구 우선 — 하지만 이상하면 다른 옵션 1개 추천 후 의견 묻기

#### 7.1 실행

도구별 명령은 LLM이 그 시점 표준 명령(예: `vercel deploy`, `netlify deploy --prod`, `cloudflared tunnel --url ...`)을 박는다. 실행 중 에러는 `error-translator`가 가로챔.

진행 가시화 (디자이너 시야 빈 화면 X):
- "회원가입·로그인 창이 떴어요" / "올리는 중이에요 (1~2분)" / "끝났어요" 3단계 안내
- 완료까지 시간 분 단위 안내

#### 7.2 결과 URL 정형 추출

도구 출력에서 URL을 정규식으로 추출 (`https://[a-zA-Z0-9.-]+\.(vercel\.app|netlify\.app|trycloudflare\.com|...)`). 추출 실패면 도구 출력 마지막 줄들 그대로 노출 + "URL 어느 줄인지 모르겠으면 알려주세요" 안내.

`SHARE_URL`, `SHARE_TOOL` 보관.

### 8. 사후 처리

#### 8.1 공유 시점 commit (`safe-save` 위임)

`Task` tool로 `safe-save` 위임. 자연어 인자:

> 외부 공유 직전 시점 commit. 메시지: "공유 — <YYYY-MM-DD HH:mm> <대상> (<INTENT 라벨>)"
>
> 예: "공유 — 2026-04-29 17:32 클라이언트 1차 시안 (B 단기)"

`safe-save`가 자동 한국어 메시지 생성·commit. push는 묻지 않고 로컬만 (외부 공유는 별 작업이라 디자이너가 명시 발화 시에만 push).

`COMMIT_SHA` 보관.

#### 8.2 §공유 이력 갱신

`CLAUDE.project.md`의 `<!-- kd:slot:share-history -->` 슬롯 안 표에 1줄 추가 (`Edit` `replace_all: false`):

```markdown
| <YYYY-MM-DD HH:mm> | <INTENT 라벨> | <대상> | <SHARE_URL> | 활성 (`<COMMIT_SHA>`) |
```

대상이 미정이면 묻지 말고 "공유" 한 단어로 (디자이너가 적은 발화에서 추출 시도, 실패 시 generic).

슬롯 자체가 없으면 (구버전 템플릿) §인계 주의사항 다음에 새로 추가.
